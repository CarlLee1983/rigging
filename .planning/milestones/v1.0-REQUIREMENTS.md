## Outcome Notes

### Validated As-Designed

大多數 requirement 如原 spec 落地。特別記錄值得 Retrospective 參考的：

- **AUTH-11 Scenario B wrap** — 研究階段假設 BetterAuth 可能自動 invalidate 其他 session，spike 結果（`03-01-spike-result.json`）證明 Scenario B：BetterAuth 1.6.5 留所有 session 活著，Rigging 需自己 wrap `revokeSessions`。ADR 0016 記錄這個 trade-off。
- **API Key prefix → hash lookup (AUTH-13)** — P3 原始實作以 prefix column 為主鍵 lookup，P4 整合測試暴露「多把 key 共用 `rig_live_` prefix」時會選錯列，P4 04-04 atomic commit 改為 `findByKeyHash(SHA-256 hex)` lookup。以 adopted-scope 模式記錄於 04-04-SUMMARY.md。

### Adjusted During Execution

- **ARCH-05 status code** — 原文寫 `ValidationError→400`，實際 P4 04-04 hardening 改為 `VALIDATION → 422`（語意更正確；Elysia `error_handler.plugin.ts` 前置分支判 `code === 'VALIDATION'` + numeric status）。上表已更新。
- **QA-02 testcontainers → shared Postgres** — 原 spec 要求 `testcontainers`，P5 05-01 執行時發現 dev velocity 折衷下改用共用 Postgres + `scripts/ensure-agent-schema.ts` 的冪等 bootstrapper；ADR 0018 記錄此 deviation + 配套（CI postgres service + quickstart 文件指引）。
- **API Key RAW_KEY_LENGTH 52 → 73** — P5 05-01 撰寫單元測試時暴露 BetterAuth api-key plugin 實際用 SHA-256 **base64url**（43 字元）而非 hex（64 字元），raw key 總長 73 而非 52。修正於 `identity-service.adapter.ts` + 5 個測試。

### Known Gaps at Close

- **Phase 04 SECURITY audit deferred** — `$gsd-secure-phase 04` 未在 milestone 範圍內執行。threat model 於 `04-PLAN.md` 存在，但 retroactive mitigation-verification 屬 `$gsd-secure-phase` workflow 職責；以 standalone task 補票。
- **CI 首跑 pending** — CI workflow 已寫入，但需 push branch + PR 才會被 GitHub Actions 實跑第一次（G22 checklist 第 10 項）。

## Out of Scope (Confirmed at v1.0 close)

以下項目在 v1.0 明確排除，close 時覆查仍成立；若 v1.1+ 要重新引入需顯式 ADR。

| Feature | Reason |
|---------|--------|
| 前端 UI / React / Vue / SvelteKit | v1 API-first；UI 留給使用 framework 的人自行搭配 |
| OAuth / SSO / 2FA / Magic Link / Passkey | 每多一條身分路徑就多一條 AuthContext 解析分支，違反「一條軌道」論述 |
| MCP server / A2A / Agent orchestration | v1 聚焦「單 Agent 在 harness 上寫程式」 |
| OpenTelemetry / distributed tracing / Prometheus metrics | 社群可用等級先；生產級 observability 留給 PROD-03 |
| Multi-tenancy / Organization / RBAC | 下個抽象層次 |
| NPM 套件發布（@rigging/core 等） | API 還會改，不宜過早定型 |
| WebSocket / SSE / real-time events | v1 只做 REST |
| GraphQL API | v1 REST + Swagger 足以展示 harness |
| 容器 image 發布 | v1 docker-compose 供 dev |
| Production-grade migration tooling | 社群可用等級 |
