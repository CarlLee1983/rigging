## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner ‚Üí ephemeral postgres service | `services.postgres` runs in a sibling Docker container on the same runner host; reachable at `localhost:5432`; lifecycle bound to job |
| GitHub Actions secrets ‚Üí workflow env | `secrets.CI_BETTER_AUTH_SECRET` injected into env; never logged; fallback string is non-secret (CI-only ephemeral) |
| Workflow YAML ‚Üí repo collaborators with write access | YAML changes can exfiltrate secrets via runner ‚Äî protected by branch protection requiring PR review |
| `git status --porcelain` ‚Üí migration drift signal | Trusts that `drizzle-kit generate` is deterministic per schema state ‚Äî verified live |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Tampering | Concurrency cancel hides flaky test signal | accept | Force-push intentionally re-runs CI; flaky tests should be quarantined per AGENTS.md, not detected via repeated CI runs |
| T-05-03-02 | Information disclosure | BETTER_AUTH_SECRET in env logs | mitigate | GitHub Actions auto-redacts `secrets.*` in logs; fallback literal is non-secret |
| T-05-03-03 | Denial of service | Concurrency cancel allows attacker to spam force-pushes | accept | GitHub rate-limits actions per repo; only collaborators can push; risk minimal |
| T-05-03-04 | Repudiation | Coverage artifact retention only 7 days | accept | Long-term audit needs are out of v1 scope; PR review at merge time captures gate verdict |
| T-05-03-05 | Spoofing | postgres service container could be MITM'd by another runner job | accept | Service containers are job-scoped; no cross-job network access; risk N/A |
| T-05-03-06 | Elevation of privilege | Unprivileged contributor opens PR with malicious workflow change | mitigate | GitHub default is `pull_request_target` is NOT used here ‚Äî `pull_request` runs in fork's context with no secret access; secrets only flow on `push` to main |
