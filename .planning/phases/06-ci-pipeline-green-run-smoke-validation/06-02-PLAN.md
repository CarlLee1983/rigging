<objective>
收集 CI-05 的 5 種 fail-mode evidence — 在一個 sacrificial PR 上 force-push 5 次，每次只製造一種最小破壞，觀察對應 gate 紅燈並把 check run URL 貼進 06-02-SUMMARY 的 5-row 表格；5 輪完成後 PR close 不 merge（D-05）。

Purpose: 驗證 Plan 1 已 land 的 pipeline 每一層 gate 都能真正擋住 bad PR。CI-05 要求「刻意破壞各 gate 並取 check URL 舉證」— 這只能透過真 GitHub Actions 的紅燈取得，不能在 local 驗證。Plan 1 已提供「綠 baseline」對照組（D-02），故本 plan 每次紅燈僅對應當次破壞，無 baseline noise。

  - sacrificial PR（建議 branch `phase-6-failmode-demo`，base=main, head=此 branch），5 次 force-push 各留一個 red CI run

  - sacrificial PR 最終 closed 不 merge；sacrificial branch 刪除（或至少不 reuse）
</objective>
