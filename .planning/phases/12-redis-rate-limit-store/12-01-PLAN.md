# Redis Rate Limit Store Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the in-memory rate limit store to a Redis-backed persistent store to ensure rate limiting state persists across restarts and multiple instances.

**Architecture:** Use `ioredis` for connection management and `@better-auth/redis-storage` to offload BetterAuth rate limit counters to Redis. Additionally, implement a global Elysia rate limit plugin that uses Redis when available.

**Tech Stack:** `ioredis`, `@better-auth/redis-storage`, `elysia-rate-limit`.

### Task 2: Redis Client Factory

**Files:**
- Create: `src/shared/infrastructure/redis/client.ts`

- [x] **Step 1: Implement createRedisClient**

```typescript
import { Redis } from 'ioredis';
import type { Logger } from 'pino';

export function createRedisClient(url: string, logger: Logger) {
  const client = new Redis(url, {
    maxRetriesPerRequest: 3,
    retryStrategy(times) {
      const delay = Math.min(times * 50, 2000);
      return delay;
    },
  });

  client.on('error', (err) => {
    logger.error({ err }, 'Redis connection error');
  });

  client.on('connect', () => {
    logger.info('Redis connected');
  });

  return client;
}
```
