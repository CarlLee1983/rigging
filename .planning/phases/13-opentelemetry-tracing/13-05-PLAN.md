  <how-to-verify>
驗證步驟（約 3-5 分鐘）：

**Step 1: 啟動 Jaeger（使用 Docker）**
```bash
docker run --rm -d \
  --name jaeger \
  -p 16686:16686 \
  -p 4318:4318 \
  jaegertracing/all-in-one:latest
```

等待約 5 秒讓 Jaeger 完全啟動。

**Step 2: 啟動 Rigging app（帶 OTLP endpoint）**
```bash
OTEL_EXPORTER_OTLP_ENDPOINT=http://localhost:4318 \
BETTER_AUTH_SECRET=$(openssl rand -hex 32) \
BETTER_AUTH_URL=http://localhost:3000 \
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/rigging \
bun run dev
```

（DATABASE_URL 用你現有的本地 Postgres 值；若 DB 無法連線，app 仍會啟動，只是 /health 會回 503）

**Step 3: 觸發幾個 HTTP 請求**
```bash
curl http://localhost:3000/health
curl http://localhost:3000/swagger
curl http://localhost:3000/not-found-route
```

**Step 4: 在 Jaeger UI 確認 traces**

打開瀏覽器：http://localhost:16686

1. 在 "Service" 下拉選單選擇 **rigging**
2. 點擊 "Find Traces"
3. 確認看到：
   - Service 名稱顯示為 "rigging"
   - Trace 列表中有 GET /health、GET /swagger 等項目
   - 點開任一 trace，確認 span attributes 中有：
     - `http.route` = `/health`（或對應 route）
     - `http.request.method` = `GET`
     - `http.response.status_code` = `200`（或其他狀態碼）
     - `http.request.duration`（毫秒數值）
   - 404 請求的 span status 應顯示為 Error

**Step 5: 清理**
```bash
docker stop jaeger
```
